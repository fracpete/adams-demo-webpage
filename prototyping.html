ADAMS strives to enable you to quickly set up and modify complex workflows, allowing you to concentrate on the functionality rather than the presentation.

Drag'n'drop of actors

Using drag'n'drop you can quickly assemble a flow:

Drag'n'drop

You simply select an actor in the Actors tab and drag it onto the actor where you want to add it relative to (beneath, after or at this exact location)

Dragging the StringConstants source onto the flow.

Drag'n'drop

After dropping the actor onto another, you select how to insert it: beneath, after or at this exact location.

Specifying how to insert the StringConstants source into the flow.

Context-sensitive adding of actors

Instead of adding actors by dragging them from the library of available ones, you can use ADAMS' context-awareness when adding actors via right-clicking on an actor and adding it beneath, here or after:

Context-sensitive adding of actors

Depending on where you are trying to add an actor (parent actor, actors before and/or after), different suggestions of commonly used actors are being offered in the combobox at the top.

You can always bring up the complete range of possible actors by clicking on the class tree button. The popup allows you to perform a search within the (class) names of the actors. While typing the search string the view gets updated and only the actors that match the search string are being displayed.

Searching for actors

The search box allows you to search in the actor class names (substring matching). Only actors that match str are being displayed in this screenshot.

For now we settle on using the SingleFileSupplier actor as source. Adding an additional actor after the SingleFileSupplier will remove the source actors from the class tree and also change the suggested actors to add next:

Context-sensitive suggestions

Adding an actor after the SingleFileSupplier changes the context and therefore the suggestions. We will choose the WekaFileReader in this example.

After adding the WekaFileReader we get presented with another list of suggested actors:

Context-sensitive suggestions

We will choose the WekaClassSelector this time, as we want to define the class attribute of the dataset that we are loading.

With the dataset fully configured, the WekaCrossValidationEvaluator actor is the next one that we want to choose from the suggestions:

Context-sensitive suggestions

With the WekaCrossValidationEvaluator we can evaluate the dataset using a globally defined classifier.

To obtain a summary of the cross-validation performed, we will add the WekaEvaluationSummary transformer which will generate textual output:

Context-sensitive suggestions

The WekaEvaluationSummary transformer turns an Evaluation object into a textual representation.

And finally, we add the Display sink for viewing the result of the cross-validation:

Context-sensitive suggestions

The Display sink is a simple actor for displaying any string (objects are turned into strings using the toString() method).

Before the flow can be executed, you still need to add a GlobalActors standalone actor before the source actor. To this special container actor, you will add (and configure) a WekaClassifier transformer. This is the transformer that the WekaCrossValidationEvaluator references.
