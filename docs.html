Overview

Less is more, that's the ADAMS philosophy in regards to the user interface. Instead to burden the user with connecting the actors in a workflow, the structure of the flow and what control actors are in use define how the data flows.

View details »

Prototyping

Simply drag actors from the large repository of available actors and drop them in the workflow where you need them. If you don't use drag'n'drop, based on the context where you are adding actors, ADAMS makes suggestions based on pre-defined rules and also filters out all actors that don't fit into this position.

View details »

Extending

ADAMS is very easy to extend and you can quickly add your own functionality. Thanks to dynamic class discovery, all you need to do is place a single class and a 16x16 icon into pre-defined packages and that's it!
